# Claude Code 프로젝트 컨텍스트

## 프로젝트 개요
이 프로젝트는 **NEET/무기력한 사람들을 위한 철학적 동반자 웹 애플리케이션**입니다. 코치나 치료사가 아닌, 진정한 **동반자** 역할을 하는 것이 핵심 철학입니다.

### 최신 변경사항 (2025-01-28)
- **미연시 스타일로 완전 전환** - 단순한 조언 앱에서 스토리 중심의 캐릭터 상호작용 게임으로 발전
- **파일 구조 분리** - 단일 HTML 파일을 HTML, CSS, JavaScript로 분리하여 관리 용이성 향상
- **3-4시간 플레이타임** - Day 1-10 구조로 체계적인 스토리 진행

### 핵심 파일
- `index.html` - 메인 HTML 구조
- `styles.css` - 모든 스타일시트
- `script.js` - JavaScript 로직 (캐릭터 시스템, 호감도, 스토리 진행)
- `philosophy_companion.html` - 구버전 (참고용, 기존 시스템)
- `README.md` - 프로젝트 전체 설명서
- `DEVELOPMENT_LOG.md` - 개발 과정 상세 기록

## 개발 완료된 주요 기능들

### ✅ 1. 캐릭터 시스템 (5명)
- **바다** (🪷) - 마음의 친구, 불교적 관점
- **퀘스트** (🔍) - 탐구의 친구, 소크라테스적 관점  
- **제로** (🌊) - 자연의 친구, 도교적 관점
- **플레임** (🔥) - 열정의 친구, 니체적 관점
- **스톤** (🗿) - 현실의 친구, 스토아 철학

### ✅ 2. 감정 일기 + 캐릭터 반응 시스템
- 사용자가 일기를 쓰면 감정 분석
- 감정에 맞는 캐릭터가 자동으로 반응
- **강화된 감정 인식**: 8가지 세분화된 감정 + 강도별 분류

### ✅ 3. 친밀도 시스템 (0-20단계)
- 만남/대화할 때마다 친밀도 증가
- 친밀도 10+ 달성 시 "소중한 친구들"에서 언제든 만남 가능
- 단계별 차별화된 대화 내용

### ✅ 4. 캐릭터 능동적 접근
- 사용자가 오랫동안 방문하지 않으면 캐릭터들이 먼저 걱정 표현
- 메인 화면에 "걱정하는 친구들" 섹션 표시

### ✅ 5. 개인적 대화 기능
- 과거 대화 내용과 감정 상태를 기억
- 시간 경과와 방문 패턴 학습
- 개인화된 대화 제공

### ✅ 6. 캐릭터와 함께 시간 보내기
- 각 캐릭터별 테마 음악과 함께하는 휴식 공간
- 3-7분 간격으로 자연스러운 ambient 메시지
- **명상/호흡 가이드 아님** - 그냥 함께 있어주는 느낌

### ✅ 7. 일상의 작은 발견 공유
- 시간대별(아침/오후/저녁) 캐릭터들의 이야기
- 친밀도에 따른 개인화된 콘텐츠

### ✅ 8. 강화된 감정 인식 시스템 (최신 완성)
```javascript
// 8가지 세분화된 감정 카테고리
- exhausted (high intensity) - 바다
- tired (medium intensity) - 바다  
- weary (low intensity) - 제로
- anxious - 퀘스트
- sad - 제로
- angry - 플레임
- confused - 스톤
- lonely - 바다
```

## 중요한 디자인 결정들

### ✅ 미연시 전환 핵심 요소 (2025-01-28 완성)
1. **스토리 중심 진행** - 24시간 복합 도서관 배경, Day 1-10 체계적 스토리
2. **캐릭터별 모에 포인트** 확정:
   - 🪷 바다: 따뜻한 큰누나 (포용력, 치유)
   - 🔥 플레임: 츤데레 작은누나 (세련, 의지력)  
   - 🌊 제로: 소꿉친구 (편안, 자연스러움)
   - 🔍 퀘스트: 키우고 싶은 동생 (순수, 호기심)
   - 🗿 스톤: 당찬 아이 (어려도 현실적 지혜)

3. **4지 선택지 + 호감도 등락** - 철학적 가치관이 맞아야 친해짐
4. **진정한 관계 추구** - 무조건적 호감이 아닌 진심이 통해야 하는 관계
5. **꿈과 현실의 경계 모호** - 인셉션 스타일 애매함으로 무한 상상 여지

### ❌ 하면 안 되는 것들 (사용자 피드백 반영)
1. **"행동 야기하기" 금지** - 명상, 호흡법, 운동 등 뭔가 "해야 할 것" 제안하면 안 됨
2. **개별 캐릭터 선택 버튼 제거** - "선택해서 만날 수 있으면 랜덤이 의미가 없다"
3. **탐구하기 메뉴 제거** - "사람들이 안 쓸 것 같다"
4. **캐릭터 모순 포인트 금지** - 스릴러 느낌 나면 안 됨, 치유 목적과 반대

### ✅ 대신 이렇게
1. **"함께 시간 보내기"** - 행동 강요 없이 그냥 같이 있어주기
2. **"오늘의 동반자 만나기"에서만 랜덤 만남** - 친밀도 쌓은 후에야 개별 만남 가능
3. **단순하고 직관적인 메뉴 구조**
4. **공익적 메시지** - 공식은 성장 중심, 팬들이 알아서 연애 2차 창작

## 기술적 특징
- **Single File Application** - 모든 코드가 `philosophy_companion.html` 하나에
- **LocalStorage 기반** - 서버 없이 완전 클라이언트 사이드
- **완전한 익명성** - 개인정보 수집 없음
- **반응형 디자인** - 모바일/데스크톱 모두 지원

## 미연시 스토리 구조 (완성됨)

### 🏗️ 도서관 배경 설정
- **3층 복합 도서관 + 호수**: 1층(카페, 정보데스크, 어린이코너), 2층(힐링라운지, 열람실), 별관(호수 관리소)
- **각 캐릭터의 자연스러운 위치**:
  - 🗿 스톤: 1층 정보데스크 (학생 알바)
  - 🔥 플레임: 1층 북카페 (츤데레 바리스타)  
  - 🔍 퀘스트: 1층 어린이코너 (다독왕 어린이)
  - 🪷 바다: 2층 힐링라운지 (자원봉사자)
  - 🌊 제로: 별관 호수 관리소 (자유로운 관리인)

### 📖 Day 1-10 스토리 진행
- **프롤로그**: 밤늦게 발견한 신비로운 도서관
- **Day 1-5**: 각 캐릭터와 첫 만남 (성격별 자연스러운 접근)
- **Day 6-8**: 관계 심화, 4지 선택지로 호감도 등락  
- **Day 9-10**: 클라이맥스, 엔딩 분기

### 🎯 엔딩 시스템
- **완벽 엔딩**: 3명 이상 호감도 15+ - "진정한 동반자들"
- **개별 특별 엔딩**: 1명 호감도 20+ - 캐릭터별 특별 아이템과 약속
- **성장 엔딩**: 평균 호감도 10+ - "새로운 시작"  
- **평범/아쉬운 엔딩**: 재플레이 유도

### 🌙 꿈과 현실의 애매함
- **현실 복귀 후 증거물들**: 각 캐릭터별 특별 아이템이 실제로 있음
- **하지만 의심 요소들**: 시간, 장소, 상황의 비논리성
- **무한 토론 거리**: 팬들이 영원히 "진짜였나 꿈이었나" 논쟁
